## 目标
- 在“索引管理”页实现：添加目录、删除目录、启用/暂停与递归开关，并将目录列表持久化到本地。
- 启动时加载已保存的目录列表；任何变更都写入配置文件。

## 技术方案
- 本地存储：使用 Tauri v2 的 `@tauri-apps/plugin-fs` 在 `appConfigDir()` 下读写 `directories.json`（若不可写，回退到 `localStorage`）。
- 目录选择：使用 `@tauri-apps/plugin-dialog` 的 `open({ directory: true, multiple: true })` 选择目录。
- 能力权限：在 `src-tauri/capabilities/default.json` 开启 `fs`（读/写 appconfig 文本文件）与 `dialog`（允许打开对话框）。

## 新增/修改文件
- 新增 `src/utils/directoriesStorage.ts`
  - `loadDirectories(): Promise<DirectoryConfig[]>`
  - `saveDirectories(dirs: DirectoryConfig[]): Promise<void>`
  - 使用 `exists/mkdir/readTextFile/writeTextFile` 与 `appConfigDir()+join('directories.json')`。
- 修改 `src/pages/IndexManagementPage.tsx`
  - 删除初始化的模拟数据；在 `useEffect` 中调用 `loadDirectories()` 设置状态。
  - `handleAddDirectory`: 通过 `dialog.open` 获取目录，去重后以默认结构追加，调用 `saveDirectories`。
  - `handleRemoveDirectory`: 从状态移除后调用 `saveDirectories`。
  - `handleToggleDirectory`: 切换 `enabled` 后调用 `saveDirectories`。
  - 递归开关（如已有 UI）同样保存。

## 插件与依赖
- 前端：安装 `@tauri-apps/plugin-fs`、`@tauri-apps/plugin-dialog`。
- Rust：在 `src-tauri/Cargo.toml` 添加 `tauri-plugin-fs = "2"`、`tauri-plugin-dialog = "2"`；在 `src-tauri/src/lib.rs` 初始化 `.plugin(tauri_plugin_fs::init())` 与 `.plugin(tauri_plugin_dialog::init())`。
- 能力权限：
  - `fs`: `fs:default`, `fs:allow-appconfig-read(+recursive)`, `fs:allow-appconfig-write(+recursive)`, `fs:allow-read-text-file`, `fs:allow-write-text-file`
  - `dialog`: `dialog:default`, `dialog:allow-open`

## 数据格式
- `directories.json`（数组）：`[{ path, enabled, recursive, lastIndexed }]`
- 路径去重规则：同一绝对路径仅保留一个条目。

## 错误处理
- 读/写失败记录到控制台并回退到 `localStorage`。
- 目录选择取消不产生变更。

## 验证
- 运行 `npm run tauri dev`，在索引管理页：

  - 删除目录与启用/暂停操作 → 刷新后状态一致。
